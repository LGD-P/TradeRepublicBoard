# Development sprints

Planned work lives in [`todo/`](todo) and moves to [`done/`](done) once shipped.
When a sprint lands, its file is moved to `done/`, dated, and the main
[`README.md`](../README.md) roadmap is updated.

Each sprint file states the **goal**, the **scope**, the **acceptance criteria**,
and any **security / privacy** notes. The project intentionally stays light:
prefer the standard library, avoid heavy dependencies, keep the CVE surface small.
